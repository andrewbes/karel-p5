Команди Karel

Програма — послідовність рядків (як у JavaScript). Рядок може закінчуватися на ; або ні. Коментарі: рядок, що починається з //.

Рух і поворот

move()
Крок на одну клітинку вперед у напрямку погляду. Якщо попереду стіна — помилка.
turnLeft()
Повернутися ліворуч на 90°.

Біпери

У Карелі є корзина: біпери, які робот зібрав з клітинок, лежать у ній; покласти біпер можна лише якщо в корзині щось є. Поле bagCount у JSON-карті задає лише початкову кількість біперів у корзині (ціле число ≥ 0; якщо поля немає — 0). Воно не пов’язане з масивом beepers: біпери на клітинках на карті й біпери в корзині рахуються окремо. Кожен успішний pickBeeper() забирає один біпер з поточної клітинки й збільшує кількість у корзині на 1. Початкове значення bagCount у JSON — не верхня межа: після збору з поля число в корзині може бути більшим за початкове (обмеження лише тим, скільки біперів на карті взагалі можна підняти).

У масиві beepers клітинку можна записати як "x,y" (один біпер) або "x,y:n", де n — скільки біперів лежить на клітинці спочатку. Якщо на клітинці більше одного біпера, над кулею показується число.

Приклад карти (фрагмент JSON):

"beepers": [
  "1,2",
  "3,4:3"
]

На (1,2) — один біпер; на (3,4) — одразу три. У демо-карті worlds/demo.json є клітинка з кількома біперами в одному записі.

putBeeper()
Покласти один біпер з корзини на поточну клітинку. Якщо корзина порожня — помилка.
pickBeeper()
Підняти один біпер з поточної клітинки в корзину (лічильник корзини збільшується на 1). Якщо на клітинці немає біпера — помилка.
beepersPresent()
Повертає true, якщо на клітинці є хоча б один біпер.
beepersInBag()
Перевірка «чи є біпери в корзині»: true, якщо в корзині є хоча б один біпер. Можна використовувати в умовах if та while (як інші предикати).
bagEmpty()
Перевірка «чи порожня корзина»: true, якщо біперів у корзині немає. Також дозволена в if / while.
bagCount()
Повертає поточну кількість біперів у корзині (ціле число). Використовується окремим рядком програми (не як умова в дужках if / while — для умов беріть beepersInBag() або bagEmpty()). Після кроку з цією командою у рядку статусу показується значення, наприклад Executed: bagCount() -> 3.

Приклад: покласти біпер лише коли він є в корзині — if (beepersInBag()) { putBeeper(); }.

Перевірки стін

frontIsClear()
true, якщо попереду немає стіни (можна зробити move()).
leftIsClear()
true, якщо зліва немає стіни.
rightIsClear()
true, якщо справа немає стіни.

Фарба

paintCorner("Колір")
Зафарбувати поточну клітинку. Колір у лапках, латиницею, наприклад: "Red", "Yellow", "Cyan" (як у демо).

Умова

if (frontIsClear()) {
  move();
}

У дужках — один із предикатів: frontIsClear(), leftIsClear(), rightIsClear(), beepersPresent(), beepersInBag(), bagEmpty(). Команда bagCount() тут не використовується — лише перелічені предикати. Тіло лише в фігурних дужках { }. Вкладені if дозволені.

Заперечення: можна поставити ! перед предикатом, наприклад while (!rightIsClear()) { move(); }.

Цикл for

for (let i = 0; i < 4; i++) {
  move();
}

Повторює тіло { } стільки разів, скільки вказано в умові i < N (N — ціле число від 0). Заголовок у стилі JavaScript: let, старт 0, порівняння <, далі будь-який коректний вираз інкременту (наприклад i++). Вкладені цикли та комбінація з if дозволені.

Структура програми
Спочатку можна оголосити нуль або більше глобальних функцій function імʼя() { … }, далі — основний код (команди, if, for, while). Глобальні функції можуть викликати одна одну.
Виклик
імʼя(); — виклик користувацької функції (не плутати з вбудованими командами на кшталт move()).
Локальні (вкладені) функції
Усередині тіла будь-якої функції (і лише там) можна оголошувати function … () { … }. Таке імʼя видно лише в цьому блоці та лише після рядка оголошення — викликати «раніше» оголошення в тому ж блоці не можна. Тіла if, for і while не додають імен назовні: локальна функція всередині циклу не видна після циклу.
Рекурсія
Дозволена: глобальна функція може викликати сама себе, локальна — себе ж у межах того ж блоку. Потрібна умова зупинки, інакше виконання не скінчиться.
Обмеження імен
У одному блоці не можна двічі оголосити функцію з тим самим імʼям (у тому числі якщо таке імʼя вже є у глобальних функціях — повтор у внутрішньому блоці теж заборонений). Імʼя функції не може збігатися з вбудованою командою (move, turnLeft, предикати, bagCount тощо) або бути зарезервованим словом: if, while, for, function, let, const, var, paintCorner. 
function walkLine() {
  if (frontIsClear()) {
    move();
    walkLine();
  }
}

Цикл while

while (frontIsClear()) {
  move();
}

Поки предикат у дужках дає true, виконується тіло { }. Ті самі умови, що й у if. Після кожного проходу тіла умова перевіряється знову. Якщо умова завжди істинна, цикл не зупиниться (перезавантажте сторінку або натисніть Reset).

Запуск

Run
Виконати всю програму з анімацією кроків.
Step
Один елемент черги: одна команда, один if (з можливим розгортанням тіла), один for (усі повтори тіла стають наступними кроками), одна перевірка while, один виклик імʼя() (у чергу додається тіло цієї функції та межі області видимості), один рядок оголошення локальної function … () { … }. Далі натискайте Step, щоб виконувати вже підставлені команди по одній. Якщо крок — це предикат (frontIsClear(), beepersInBag() тощо) або bagCount(), у рядку статусу з’являється результат після стрілки, наприклад -> true або -> 5.
Reset
Скинути світ і чергу виконання.
